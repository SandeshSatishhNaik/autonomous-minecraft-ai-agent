# 🧠 Self-Learning System: How the Bot Learns Like a Human

> This document defines the complete self-learning architecture for the Minecraft AI Agent.
> The goal: the bot should learn ALL skills in Minecraft through experience, just like a human player.

---

## 📌 The Problem with the Current Plan

The current Phase 5 (Skill Library) is a **simple lookup table**:
- Bot does something successfully → save the action sequence → replay it next time

This is NOT learning. This is **recording**. A human doesn't just record actions — they:
1. Try things and fail
2. Understand WHY they failed
3. Adjust their approach
4. Build mental models of how the world works
5. Transfer knowledge from one situation to another
6. Get progressively better at difficult tasks
7. Discover things they weren't told about
8. Set their own goals to improve

---

## 🏗️ The 5 Pillars of Self-Learning

### Pillar 1: Experience Memory (Episodic Memory)

**What:** Remember everything that happens — not just successes, but failures too.

Every action the bot takes gets saved to `data/experience_log.json`:

```json
{
  "id": "exp_001",
  "timestamp": 1711800000,
  "situation": {
    "health": 12,
    "timeOfDay": "night",
    "nearbyEntities": ["zombie", "skeleton"],
    "nearbyBlocks": ["stone", "dirt"],
    "inventory": ["wooden_sword", "3x dirt"],
    "currentGoal": "survive the night"
  },
  "action": { "type": "fight", "target": "zombie" },
  "result": "failure",
  "outcome": {
    "healthAfter": 4,
    "died": false,
    "itemsLost": [],
    "itemsGained": [],
    "error": "Took too much damage fighting two mobs at once"
  },
  "lesson": "Don't fight multiple mobs at night with low health and a wooden sword"
}
```

**Key design:**
- Log EVERY action — successes AND failures
- Include full context (what was happening, what the bot had, what it was trying to do)
- The `lesson` field is generated by asking the LLM: *"What did you learn from this?"*

---

### Pillar 2: Reflection Engine (Self-Evaluation)

**What:** Periodically stop and think about what happened. Like a human thinking before sleep: *"What went well today? What went badly?"*

**How it works:**

Every 30 minutes of play time (or after death, or after completing a major goal), the bot enters a **reflection phase**:

1. Pull the last N experiences from the log
2. Send them to the LLM with the prompt:

```
Here are your recent experiences:
[experience_1, experience_2, ...]

Reflect on the following:
1. What patterns do you see in your failures?
2. What strategies worked well?
3. What should you do differently next time?
4. What new rules should you add to your behavior?
5. Rate your overall performance (1-10) and explain why.

Output as JSON:
{
  "patterns_noticed": [...],
  "successful_strategies": [...],
  "improvement_areas": [...],
  "new_rules": [...],
  "self_rating": 7,
  "rating_reason": "..."
}
```

3. Save the reflection to `data/reflections.json`
4. Inject relevant reflections into future prompts

**Example reflection output:**
```json
{
  "patterns_noticed": [
    "I keep dying at night because I don't prepare shelter before sunset",
    "Mining without torches leads to getting ambushed by mobs"
  ],
  "successful_strategies": [
    "Gathering wood immediately after spawn works well",
    "Eating before combat improves survival rate"
  ],
  "improvement_areas": [
    "Start building shelter when timeOfDay is 'afternoon', not 'evening'",
    "Always craft torches before going underground"
  ],
  "new_rules": [
    "RULE: If time is afternoon and no shelter exists → build shelter immediately",
    "RULE: Never enter caves without at least 8 torches"
  ],
  "self_rating": 5,
  "rating_reason": "Died twice to mobs at night. Need to prioritize shelter earlier."
}
```

### Pillar 3: Strategy Evolution (Adaptive Behavior)

**What:** The bot's behavior changes over time based on what it has learned.

**How it works:**

The bot maintains a `data/learned_rules.json` file — a growing list of behavioral rules discovered through experience:

```json
{
  "rules": [
    {
      "id": "rule_001",
      "source": "reflection_after_death_003",
      "rule": "Build shelter before sunset, not after",
      "confidence": 0.9,
      "times_applied": 12,
      "times_helped": 10,
      "created": "2026-04-02",
      "category": "survival"
    },
    {
      "id": "rule_002",
      "source": "reflection_005",
      "rule": "Mine iron before attempting to explore caves",
      "confidence": 0.7,
      "times_applied": 5,
      "times_helped": 4,
      "created": "2026-04-03",
      "category": "resource_gathering"
    },
    {
      "id": "rule_003",
      "source": "death_analysis_007",
      "rule": "Never fight Creepers in melee — use bow or run",
      "confidence": 1.0,
      "times_applied": 8,
      "times_helped": 8,
      "created": "2026-04-04",
      "category": "combat"
    }
  ]
}
```

**Key mechanics:**
- **Confidence scoring:** Each rule starts with confidence 0.5. Every time it helps → +0.1. Every time it fails → -0.2. Rules below 0.2 get deleted (wrong lessons).
- **Rule injection:** Before each LLM call, inject the top 5 most relevant rules into the prompt.
- **Rule evolution:** Rules can be **updated** when the bot discovers a better version (e.g., "build shelter before sunset" → "build shelter by noon to have time for furnace").

**How XP is earned:**
- Successfully completing an action → +1-3 XP (depending on difficulty)

- Beating personal best metric → +3 XP bonus

**How levels affect behavior:**
- At Lv0: Bot relies fully on LLM for decisions
- At Lv1-2: Bot uses saved sequences + LLM
- At Lv3-4: Bot mostly uses learned rules, LLM for edge cases only
- At Lv5: Bot executes skill autonomously without LLM call (saves tokens)

### Pillar 5: Curiosity & Discovery Engine

**What:** The bot doesn't just survive — it actively explores and discovers new things.

**How it works:**

The bot maintains a `data/discoveries.json` — things it has learned about the Minecraft world through experience:

```json
{
  "discoveries": [
    {
      "id": "disc_001",
      "what": "Iron ore is found between Y:0 and Y:63",
      "how_learned": "Mined at different Y levels, found iron only below Y:63",
      "confidence": 0.8,
      "useful_for": ["mining", "crafting"]
    },
    {
      "id": "disc_002",
      "what": "Creepers don't attack during the day in open areas",
      "how_learned": "Observed creeper behavior over 5 daytime encounters",
      "confidence": 0.6,
      "useful_for": ["combat", "survival"]
    },
    {
      "id": "disc_003",
      "what": "Cooking raw beef gives 8 hunger points vs 3 raw",
      "how_learned": "Compared hunger gained from raw vs cooked meat",
      "confidence": 1.0,
      "useful_for": ["farming", "survival"]
    }
  ]
}
```

**Curiosity triggers:**
1. **Unknown block:** Bot sees a block type it hasn't interacted with → try to mine it
2. **Unknown item:** Bot gets an item it hasn't used → try to craft with it or use it
3. **Unexplored area:** Bot hasn't been to a biome type → explore it
4. **Failed craft:** Bot tries to craft something and fails → experiment with different recipes
5. **New mob:** Bot encounters a mob type for the first time → observe before engaging

**Exploration goals (auto-generated):**
```
"I've never been to a desert biome. Let me explore in that direction."
"I found a dungeon but haven't entered one yet. Let me try."
"I've never used an enchanting table. Let me figure out how."
"I've been mining at Y:30. What happens if I go to Y:10?"
```

## 🔄 How It All Works Together (Learning Loop)

```
1. ACT        → Bot takes an action in the world
2. OBSERVE    → Record the full situation + result
3. EVALUATE   → Was it a success or failure? Why?
4. LEARN      → Extract a lesson / rule / discovery
5. REMEMBER   → Save to experience log + rules + skills
6. REFLECT    → Periodically review recent experiences
7. ADAPT      → Update behavior rules and strategies
8. IMPROVE    → Skill level increases, unlocking new abilities
9. EXPLORE    → Curiosity drives discovery of new things
10. REPEAT    → Smarter each cycle
```

## 🧬 Integration Points

| Existing System | How Learning Connects |
|---|---|
| **Agentic Loop (Phase 4)** | Every tick → log experience → check rules before acting |
| **Skill Library (Phase 5)** | Replaced by Skill Progression System (Pillar 4) |
| **Critic Node (Phase 4)** | On failure → log experience → extract lesson → update rules |
| **Goal Engine (Phase 6)** | Uses skill levels to set appropriate difficulty goals |
| **Prompt Builder (Phase 2)** | Injects top rules + skill levels + recent reflections |
| **Death Recovery (Phase 7)** | On death → run reflection → learn from death |
| **Dashboard (Phase 3.5)** | Display skill levels, rule count, learning progress |

## 🎯 What This Enables

| Without Self-Learning | With Self-Learning |
|---|---|
| Bot repeats same mistakes | Bot never makes the same mistake twice |
| Static behavior forever | Behavior evolves over days of play |
| Needs human-written rules | Discovers rules through experience |
| Can only do pre-programmed skills | Discovers new techniques on its own |
| LLM decides everything fresh | Most decisions come from learned rules (faster, cheaper) |
| No concept of "getting better" | Measurable skill progression with levels |
| No curiosity or exploration drive | Actively seeks new experiences |
| Forgets between sessions | Persistent knowledge across ALL sessions |
